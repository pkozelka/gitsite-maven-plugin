gitsite:deploy

Full name:

net.kozelka.maven:gitsite-maven-plugin:0.1-SNAPSHOT:deploy

Description:

Pushes prepared static site into a special git branch. Can be used to simplify maven site publication on GitHub Pages.

Attributes:

  • Requires a Maven project to be executed.
  • Binds by default to the lifecycle phase: site-deploy.

Optional Parameters

Name Type Since Description
commitMessage String - Commit message to be used for publishing. This is passed to String.format() where first parameter is number of published files; therefore, you can use '%d' placeholder to reference it.
Default value is: Publishing ${project.artifactId} site with %d files.
User property is: gitsite.commitMessage.
gitBranch String - Which branch to put the generated site into.
Default value is: gitsite.
User property is: gitsite.branch.
gitScmUrl String - The SCM URL where the site will be deployed. By default we reuse the current project's scm.
Default value is: ${project.scm.developerConnection}.
User property is: gitsite.gitScmUrl.
inputDirectory File - The site directory to be deployed. It must be already completely generated.
Default value is: ${project.build.directory}/staging.
User property is: gitsite.inputDirectory.
keepHistory boolean - Whether to keep previous commits in the site branch. If false, all previous commits will be deleted with each update.

DANGEROUS: when false, the gitBranch will be replaced with single commit, and all its history will be dropped. While this is useful for saving space, you should really make sure that you use a dedicated branch name (and not a code branch)!!!


Default value is: true.
User property is: gitsite.keepHistory.
logfile File - Where to store log of shell executions.
Default value is: ${project.build.directory}/gitsite-deploy.log.
roots String - Comma-separated list of roots for subdirs.
Default value is: VERSION,BRANCH.
User property is: gitsite.roots.
subdir String - Subdirectory for placing the site's content.

Typically, we place content on the root. However, in some cases, we need to keep multiple static sites under one git-based hosting. Sample use-cases are multiple versions and Bitbucket hosting.


Default value is: ..
User property is: gitsite.subdir.
